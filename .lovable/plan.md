

## الخطة: عرض النتائج بحسب الفلاتر المختارة

### المشكلة
في صفحة `/admin/content`:
- قائمة المواد المنسدلة تعرض دائماً إجمالي الدروس/الأسئلة لكل المواد (جميع الصفوف)، بغض النظر عن فلتر الصف.
- العنوان أعلى الصفحة يعرض فقط `{filteredLessons.length} درس مشترك` بدون توضيح المادة أو الصف المختار.
- لا يوجد عداد للأسئلة في النتائج المفلترة.

### الحل (ملف واحد فقط: `src/pages/admin/AdminContent.tsx`)

#### 1) ربط عدّاد قائمة المواد بفلتر الصف
السطر 832-839 — عند بناء عدّاد كل مادة، يتم تطبيق `filterGradeLevel` أيضاً:
```ts
const subjectLessons = scopedLessons.filter(l =>
  l.subject_id === s.id &&
  (!filterGradeLevel || l.grade_level === Number(filterGradeLevel))
);
```
بحيث "كيمياء (12 درس، 80 سؤال)" يصبح "كيمياء (4 دروس، 25 سؤال)" عند اختيار "أول ثانوي".

#### 2) إضافة شريط ملخص نتائج البحث أسفل الفلاتر
بين سطر 851 و 853، إضافة بطاقة صغيرة (`Card`) تعرض ديناميكياً:
- اسم المادة المختارة (أو "جميع المواد")
- اسم الصف المختار (أو "جميع الصفوف")
- عدد الدروس المطابقة (`filteredLessons.length`)
- عدد الأسئلة المطابقة (محسوبة من `questions.filter(q => filteredLessons.some(l => l.id === q.lesson_id)).length`)

مثال على الشكل:
```
نتائج البحث: مادة الكيمياء، الصف الأول الثانوي
4 دروس · 25 سؤال
```

#### 3) تحديث العنوان أعلى الصفحة (سطر 817)
استبدال `{filteredLessons.length} درس مشترك` بنص يعكس الفلاتر:
```
{filteredLessons.length} درس · {filteredQuestionsCount} سؤال
```

### النطاق
- ملف واحد فقط: `src/pages/admin/AdminContent.tsx`
- لا تغييرات في قاعدة البيانات أو منطق الجلب
- `filteredQuestionsCount` يُحسب محلياً من بيانات موجودة سلفاً (`questions` و `filteredLessons`) — صفر استعلامات إضافية

### النتيجة
المسؤول يرى فوراً تأثير كل فلتر يختاره، مع توضيح بصري لمحتوى النتيجة (المادة + الصف + الأعداد) بدلاً من رؤية أرقام إجمالية مضللة.
