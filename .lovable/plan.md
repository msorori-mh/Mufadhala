
## التحليل

- في `ModeSelector.tsx`: عندما `duration_minutes` غير محدد أو 0، يتم تعطيل بطاقة الامتحان الصارم بالكامل وتظهر رسالة "يرجى التواصل مع الإدارة".
- `StrictMode.tsx` يستخدم `model.duration_minutes * 60` ثابتاً كمدة الاختبار.

## الحل

السماح للطالب باختيار المدة بنفسه عندما لا يحدد الأدمن مدة. الحد الأدنى **30 دقيقة**، مع اقتراحات سريعة (30/45/60/90/120 دقيقة).

### 1. `src/pages/past-exam/ModeSelector.tsx`
- إزالة منطق التعطيل عند عدم وجود مدة. البطاقة دائماً قابلة للنقر.
- إذا `hasDuration = true` → نفس السلوك الحالي (استخدام مدة الأدمن).
- إذا `hasDuration = false` → عند الضغط على "ابدأ الامتحان الصارم" يفتح حوار جديد لاختيار المدة:
  - عرض أزرار سريعة: 30، 45، 60، 90، 120 دقيقة.
  - حقل إدخال يدوي (Input number) بقيمة افتراضية 60 وحد أدنى 30.
  - زر "متابعة" يعطّل إذا كانت القيمة < 30.
  - بعد الاختيار → تخزين المدة في state وفتح حوار التحذير الموجود حالياً.
- تحديث نص العرض في بطاقة المؤقت من "—" إلى "اختر المدة".
- إزالة رسالة "هذا النموذج لا يحتوي على مدة محددة...".
- إضافة badge صغيرة "أنت تحدد المدة" بدل الرسالة المعطّلة.

### 2. تمرير المدة المختارة إلى `StrictMode`
- تعديل `PastExamPractice.tsx`: حالة جديدة `selectedDuration` تُمرَّر لـ `StrictMode` كـ prop.
- `StrictMode` يستخدم: `effectiveDuration = model.duration_minutes ?? selectedDuration`.

### 3. ملفات معدَّلة

| الملف | التغيير |
|---|---|
| `src/pages/past-exam/ModeSelector.tsx` | حوار اختيار مدة + إلغاء التعطيل + تمرير المدة عبر callback |
| `src/pages/past-exam/StrictMode.tsx` | قبول prop `customDurationMinutes` واستخدامه عند عدم وجود `duration_minutes` |
| `src/pages/PastExamPractice.tsx` | state للمدة المخصصة + تمريرها للمكونات |

## النتيجة

- النماذج التي حدد لها الأدمن مدة → تعمل كما هي.
- النماذج بدون مدة → الطالب يختار مدة (30 دقيقة كحد أدنى) ويبدأ الامتحان مباشرة.
- لا حاجة لانتظار الإدارة.
