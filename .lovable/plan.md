

# قائمة متطلبات نشر Google Play — تقرير التوافق

## الهدف
إنشاء ملف PDF شامل يوثق حالة التوافق الحالية لتطبيق مُفَاضَلَة مع متطلبات Google Play، مقسم إلى أقسام واضحة مع حالة كل بند (✅ متوفر / ⚠️ يحتاج عمل / ❌ غير متوفر).

## ملخص التدقيق

### 1. المتطلبات التقنية (The Build)
| البند | الحالة |
|-------|--------|
| صيغة AAB | ⚠️ يتم عبر `bundleRelease` محلياً — لا تغيير في الكود |
| Target SDK 34+ | ⚠️ يُضبط في `build.gradle` المحلي بعد `cap add android` |
| حجم < 200MB | ✅ التطبيق خفيف (Capacitor WebView) |
| توقيع رقمي | ⚠️ يتم محلياً بـ Upload Key — لا تغيير في الكود |
| Package Name | ✅ `com.mufadhala.app` فريد ومضبوط |

### 2. المتطلبات القانونية والخصوصية
| البند | الحالة |
|-------|--------|
| سياسة الخصوصية | ✅ موجودة `/privacy-policy` — تحتاج رابط دائم عام |
| حذف الحساب | ✅ صفحة `/delete-account` موجودة في التطبيق |
| حذف الحساب من المتصفح | ⚠️ يجب توفير صفحة ويب مستقلة (ليست داخل التطبيق فقط) |
| أمان البيانات (Data Safety) | ⚠️ يُعبأ يدوياً في Google Play Console |

### 3. محتوى المتجر (Store Listing)
| البند | الحالة |
|-------|--------|
| اسم التطبيق (≤30 حرف) | ✅ "مُفَاضَلَة" (8 أحرف) |
| وصف قصير (≤80 حرف) | ❌ يحتاج كتابة |
| وصف كامل (≤4000 حرف) | ❌ يحتاج كتابة |
| أيقونة 512×512 | ✅ `icon-512.png` موجودة |
| Screenshots | ❌ يحتاج إعداد يدوي |
| Feature Graphic 1024×500 | ❌ يحتاج تصميم |

### 4. معايير الجودة
| البند | الحالة |
|-------|--------|
| ليس WebView فقط | ✅ يقدم قيمة مضافة (اختبارات، ذكاء اصطناعي، إنجازات) |
| حقوق محتوى | ✅ محتوى أصلي |
| الإعلانات | ✅ لا توجد إعلانات |
| أيقونات مخصصة | ⚠️ يجب التحقق من استبدال أيقونات Capacitor الافتراضية |

## التغييرات المطلوبة في الكود

### ملف واحد: إنشاء ملف PDF بالنتائج
- إنشاء ملف PDF عربي شامل في `/mnt/documents/` يحتوي على كل البنود أعلاه مع تعليمات تفصيلية لكل بند يحتاج عمل
- يتضمن نصوص جاهزة للوصف القصير والكامل للمتجر
- يتضمن قائمة البيانات المطلوبة لنموذج Data Safety

### لا تغييرات في كود المشروع
جميع البنود المعلقة (AAB, SDK, توقيع) تتم محلياً أو في Google Play Console — لا تعديلات على ملفات المشروع.
