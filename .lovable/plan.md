
## المشكلة
في صفحة `/admin/students`، الفلاتر المتسلسلة تعمل جزئياً:
- ✅ فلتر الكلية يتقيّد بالجامعة المختارة (موجود)
- ✅ فلتر التخصص يتقيّد بالكلية المختارة (موجود)
- ❌ **لكن**: عند اختيار الجامعة فقط (بدون كلية)، يظهر فلتر التخصص كل التخصصات بدلاً من تخصصات كليات تلك الجامعة فقط.

## الحل
ملف واحد: `src/pages/admin/AdminStudents.tsx`

تعديل منطق `filterMajorsList` (السطر 117-119) ليتسلسل بالكامل:

```ts
const filterMajorsList = filterCollegeId
  ? majors.filter((m) => m.college_id === filterCollegeId)
  : filterUniversityId
    ? majors.filter((m) => 
        colleges.some((c) => c.id === m.college_id && c.university_id === filterUniversityId)
      )
    : majors;
```

**القاعدة الجديدة**:
- لا جامعة ولا كلية → كل التخصصات
- جامعة فقط → تخصصات كل كليات تلك الجامعة
- كلية محددة → تخصصات تلك الكلية فقط

## ما لن يُمَس
- خيار "الكل" يبقى موجوداً في كل الفلاتر (السطور 260, 272, 283, 294, 305).
- منطق إعادة التعيين عند تغيير الجامعة/الكلية يبقى كما هو.
- نموذج التعديل (Edit Dialog) يبقى دون تغيير.
