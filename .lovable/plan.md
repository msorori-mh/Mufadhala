

## إصلاح: منع التسجيل المكرر بنفس رقم الجوال

### المشكلة
دالة `register-student` الحالية عند العثور على طالب مسجل بنفس الرقم، تقوم بتحديث بياناته وتسجيل دخوله تلقائياً بدلاً من رفض التسجيل. هذا يسمح لأي شخص بالكتابة فوق بيانات طالب آخر.

### الحل

**ملف واحد**: `supabase/functions/register-student/index.ts`

تعديل الجزء الخاص بـ "RETURNING USER" (سطر 173-212):
- بدلاً من تحديث بيانات الطالب وتسجيل دخوله، يتم إرجاع خطأ واضح:
  ```
  "هذا الرقم مسجل مسبقاً. يرجى تسجيل الدخول بدلاً من إنشاء حساب جديد."
  ```
- كود الحالة: `409` (Conflict)
- حذف كامل منطق تحديث البيانات وتسجيل الدخول للمستخدم الموجود

### على جانب الواجهة (Register.tsx)
- عند استلام رسالة "مسجل مسبقاً"، عرض تنبيه مع زر يوجه المستخدم لصفحة `/login`

### التفاصيل التقنية
```
// السطر 173 - بدلاً من تحديث وتسجيل دخول
if (existingStudent) {
  return jsonResponse({ 
    error: "هذا الرقم مسجل مسبقاً. يرجى تسجيل الدخول بدلاً من إنشاء حساب جديد." 
  }, 409);
}
```

هذا يحمي بيانات الطلاب المسجلين ويوجه المستخدم الموجود لصفحة تسجيل الدخول.
