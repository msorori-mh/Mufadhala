

# خطة: بناء APK مستقل لا يعتمد على Lovable

## المشكلة الحالية
التطبيق حالياً يعتمد على سيرفر Lovable Preview (`lovableproject.com`) لتحميل الواجهة. هذا يعني أن التطبيق يحتاج اتصال إنترنت ويعتمد على بنية Lovable التحتية.

## الحل
تعديل إعدادات Capacitor لتحميل ملفات الواجهة المبنية محلياً من داخل التطبيق (مجلد `dist`)، بدلاً من تحميلها من الإنترنت.

## التغييرات المطلوبة

### 1. تعديل `capacitor.config.ts`
- **حذف** قسم `server.url` بالكامل حتى يقرأ التطبيق الملفات من مجلد `dist` المحلي
- إبقاء باقي الإعدادات كما هي

### 2. بناء المشروع وتجهيز APK
بعد التعديل، ستحتاج تنفيذ هذه الخطوات على جهازك:

```text
1. git pull                          ← سحب آخر التغييرات
2. npm install                       ← تثبيت المكتبات
3. npm run build                     ← بناء ملفات الواجهة في dist/
4. npx cap sync android              ← نسخ الملفات للمشروع الأندرويد
5. npx cap open android              ← فتح Android Studio
6. Build → Build APK من Android Studio
```

## ملاحظة مهمة
- التطبيق سيعمل بدون Lovable لكنه **لا يزال يحتاج اتصال إنترنت** للوصول لقاعدة البيانات والمصادقة (Lovable Cloud)
- الفرق: الواجهة (HTML/CSS/JS) ستكون محملة داخل التطبيق مباشرة = أسرع في الفتح ولا يعتمد على سيرفر Lovable للعرض
