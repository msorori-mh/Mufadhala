

# خطة: تحسين تصنيف المحتوى حسب المادة الدراسية

## ملخص
ربط عرض وإدارة المحتوى التعليمي بالمواد الدراسية (Subjects)، بحيث يتم فلترة المحتوى حسب المادة وعرضه فقط للتخصصات/الكليات/الجامعات المرتبطة بها.

## التغييرات المطلوبة

### 1. صفحة المحتوى (AdminContent.tsx) - إضافة فلتر المادة

- إضافة **فلتر ثالث للمادة** في شريط الفلاتر (بعد الجامعة والكلية)
- عند اختيار مادة معينة، تظهر فقط الدروس المرتبطة بها
- قائمة المواد المتاحة في الفلتر تتغير ديناميكياً بحسب الكلية/الجامعة المختارة (تعتمد على جدول `major_subjects` لمعرفة أي مواد مرتبطة بتخصصات تلك الكلية)

### 2. نموذج إضافة/تعديل الدرس - تحسين حقل المادة

- حقل المادة موجود حالياً لكنه يعتمد على `lessonMajorId` الذي نادراً ما يُستخدم
- تعديل منطق الفلترة: عند اختيار كلية، تظهر المواد المرتبطة بتخصصات تلك الكلية عبر `major_subjects`
- إذا لم يتم اختيار كلية، تظهر جميع المواد

### 3. نموذج الاستيراد (Import Dialog) - إضافة حقل المادة

- إضافة حقل اختياري لتحديد المادة عند الاستيراد الجماعي
- المادة المحددة تُطبق على جميع الدروس المستوردة (مع إمكانية التحديد في الملف أيضاً)

### 4. صفحة المواد (AdminSubjects.tsx) - بدون تغيير جوهري

- النظام الحالي للربط بالتخصصات يعمل بشكل صحيح عبر `major_subjects`
- لا حاجة لتغييرات في هذه الصفحة

## التفاصيل التقنية

**الملفات المتأثرة:**
- `src/pages/admin/AdminContent.tsx` - التعديل الرئيسي

**المنطق:**
```text
المادة ← مرتبطة بتخصصات عبر major_subjects
التخصص ← تابع لكلية
الكلية ← تابعة لجامعة

عند فلترة بمادة: أظهر الدروس التي subject_id = المادة المختارة
عند فلترة بكلية: أظهر المواد المرتبطة بتخصصات تلك الكلية
```

**لا توجد تغييرات على قاعدة البيانات** - الهيكل الحالي (`subjects`, `major_subjects`, `lessons.subject_id`) يدعم هذا بالكامل.
