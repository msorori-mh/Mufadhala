## المشكلة

خاصية الاستماع الصوتي الحالية في `SummaryTTS.tsx` تعتمد على Web Speech API المدمج في المتصفح/WebView. على أندرويد خاصة، الأصوات العربية المتاحة محدودة جداً ومكسّرة، والجودة لا يمكن السيطرة عليها لأنها تعتمد على محرك الجهاز.

## البديل الواضح والنقي

**ElevenLabs TTS** عبر Edge Function — يولّد ملف MP3 عربي طبيعي وواضح، نفس الجودة على الويب والأندرويد.

## الخيارات المتاحة

أحتاج قراراً واحداً قبل التنفيذ:

### الخيار أ: استبدال بـ ElevenLabs (الأفضل جودة)

- إنشاء Edge Function `tts-summary` تستدعي ElevenLabs API
- صوت عربي طبيعي وواضح (مثل صوت Sarah أو Charlotte بنموذج `eleven_multilingual_v2`)
- تشغيل MP3 عبر `<audio>` في المتصفح/التطبيق
- **يتطلب**: مفتاح `ELEVENLABS_API_KEY` (يجب على المستخدم تزويده)
- **التكلفة**: حسب عدد الأحرف المُولَّدة شهرياً

### الخيار ب: حذف الميزة كلياً

- إزالة `SummaryTTS` من صفحة الدرس بالكامل
- لا تكاليف، تجربة أبسط

## التوصية

الخيار (أ) — صوت ElevenLabs احترافي ويناسب منصة تعليمية. لتقليل التكلفة:

- **Cache على السيرفر**: نخزّن MP3 لكل ملخص درس مرة واحدة في Supabase Storage، فلا نعيد توليده لطلاب آخرين.
- **حد للمستخدم المجاني** (اختياري): مثل 3 استماعات/يوم، اشتراك لاستماع غير محدود.

## خطة التنفيذ (إذا اخترت الخيار أ)

### 1) Edge Function `tts-summary`

- المدخل: `{ lessonId, text }`
- التحقق من cache في `lesson_audio_cache` (جدول جديد بسيط: `lesson_id`, `audio_path`, `created_at`)
- إن لم يوجد: استدعاء ElevenLabs → رفع MP3 لـ Storage bucket `lesson-audio` → تخزين المسار
- المخرج: signed URL للـ MP3

### 2) جدول وStorage Bucket جديدان

- جدول `lesson_audio_cache` مع RLS (قراءة لجميع المصادَق عليهم، كتابة من service_role فقط)
- Bucket `lesson-audio` خاص، signed URL ساعة واحدة

### 3) إعادة كتابة `SummaryTTS.tsx`

- زر واحد: "استماع للملخص"
- عند الضغط: يستدعي الـ Edge Function، يحصل على رابط MP3، يُشغّله عبر `<audio>` HTML5
- مؤشر تحميل + زر إيقاف + التحكم في السرعة (1x / 1.25x / 1.5x عبر `audio.playbackRate`)

### 4) Secret مطلوب

سأطلب منك إدخال `ELEVENLABS_API_KEY` بعد الموافقة.
