

## خطة معدّلة: سعر افتراضي مستقل + خصم لكل منطقة

### الفكرة
بدلاً من سعر افتراضي واحد مشترك، يُدخل الأدمن **سعراً افتراضياً خاصاً بكل منطقة** + **نسبة خصم لكل منطقة**، ويُحسب السعر النهائي تلقائياً.

### التغييرات

**1. Migration — إضافة أعمدة السعر الافتراضي لكل منطقة**
- إضافة `default_price_zone_a numeric default 0` و `default_price_zone_b numeric default 0` لجدول `subscription_plans`
- نقل قيم `price_default` الحالية إليهما كقيم أولية

**2. تعديل `AdminSubscriptionPlans.tsx`**

واجهة التعديل (Dialog) — قسم الأسعار عند عدم كونها مجانية:

```text
┌─────────────────────────────────────────────────┐
│ المنطقة أ                                       │
│ السعر الافتراضي: [____]  خصم: [__]%             │
│ السعر بعد الخصم: 5,600 YER (تلقائي)            │
├─────────────────────────────────────────────────┤
│ المنطقة ب                                       │
│ السعر الافتراضي: [____]  خصم: [__]%             │
│ السعر بعد الخصم: 2,100 YER (تلقائي)            │
└─────────────────────────────────────────────────┘
```

- إزالة حقل `price_default` المشترك واستبداله بحقلين مستقلين
- حساب تلقائي: `price_zone = default_price_zone * (1 - discount / 100)`
- تحديث الجدول الرئيسي ليعرض لكل منطقة: السعر الافتراضي + نسبة الخصم + السعر النهائي
- تحديث `handleSave` لحفظ الأعمدة الجديدة

**3. تعديل `Subscription.tsx`**
- تحديث عرض الأسعار للطالب ليستخدم `default_price_zone_a/b` بدلاً من `price_default`

### الملفات المتأثرة
1. Migration SQL جديد (إضافة `default_price_zone_a`, `default_price_zone_b`)
2. `src/pages/admin/AdminSubscriptionPlans.tsx`
