
# خطة إعادة هيكلة تجربة التسجيل والدخول

## الفكرة الأساسية
تحويل التطبيق من نظام تسجيل دخول تقليدي (إيميل/باسورد/OTP) إلى تجربة مبسطة: الطالب يفتح التطبيق → يرى صفحة الميزات (Landing Page) → يضغط "ابدأ الآن" → يملأ بياناته → يتم إنشاء حسابه تلقائياً مع تحقق OTP لمرة واحدة عبر رقم الجوال.

## ملاحظة مهمة قبل التنفيذ
النظام الحالي يعتمد بالكامل على نظام المصادقة (Auth) لربط الطالب بالاشتراكات، سجل الاختبارات، الدفعات، ولوحة الصدارة. **لا يمكن إلغاء نظام الحسابات نهائياً** وإلا ستفقد كل هذه الميزات. الحل الأمثل هو **تبسيط التجربة مع الإبقاء على حساب حقيقي خلف الكواليس**.

## التغييرات المقترحة

### 1. إعادة تصميم الصفحة الرئيسية (Index)
- تبقى كما هي بعرض ميزات التطبيق
- زر "ابدأ الآن" يوجه إلى نموذج التسجيل المبسط بدلاً من صفحة تسجيل الدخول

### 2. إنشاء صفحة تسجيل مبسطة جديدة (`/register`)
- نموذج واحد يحتوي على:
  - الاسم الأول + اللقب (الاسم الرابع)
  - رقم الجوال (مع التحقق من الصيغة اليمنية)
  - المحافظة (قائمة منسدلة)
  - الجامعة → الكلية (فلترة متتالية)
- عند الإرسال: يتم إنشاء حساب عبر رقم الجوال + OTP (مرة واحدة فقط للتحقق)
- بعد التحقق: يتم حفظ البيانات وتوجيه الطالب مباشرة إلى لوحة التحكم
- الجلسة تبقى محفوظة (لا يحتاج تسجيل دخول مرة ثانية)

### 3. تبسيط تسجيل الدخول للمستخدمين الحاليين
- صفحة `/login` تصبح مبسطة: فقط إدخال رقم الجوال + OTP
- بدون إيميل أو باسورد نهائياً
- رابط صغير "لديك حساب؟ سجل دخول" في صفحة التسجيل

### 4. إلغاء الصفحات والمكونات غير المطلوبة
- إلغاء صفحة `/complete-profile` (البيانات تُجمع أثناء التسجيل)
- إلغاء صفحة `/register` القديمة واستبدالها بالجديدة
- تبسيط `AuthContext` وإلغاء التحقق من اكتمال الملف الشخصي
- إلغاء `resolveAuthDestination` profile check

### 5. تحديث التوجيه (Routing)
- `/` → صفحة الميزات (Landing)
- `/register` → نموذج التسجيل المبسط
- `/login` → تسجيل دخول برقم الجوال فقط
- بعد التسجيل/الدخول → `/dashboard` مباشرة

## التفاصيل التقنية

### الملفات المتأثرة:
- `src/pages/Index.tsx` — تعديل زر "ابدأ الآن" للتوجيه إلى `/register`
- `src/pages/Register.tsx` — إعادة بناء كامل بنموذج التسجيل المبسط
- `src/pages/Login.tsx` — تبسيط لرقم الجوال + OTP فقط
- `src/pages/CompleteProfile.tsx` — حذف أو تحويل لإعادة توجيه
- `src/contexts/AuthContext.tsx` — إزالة منطق التحقق من الملف الشخصي
- `src/hooks/useAuth.ts` — إزالة profile check
- `src/lib/authRouting.ts` — تبسيط التوجيه
- `src/App.tsx` — تحديث Routes

### آلية العمل خلف الكواليس:
1. الطالب يملأ النموذج → يُرسل OTP لرقم الجوال
2. يُدخل رمز التحقق → يتم إنشاء حساب في نظام المصادقة برقم الجوال
3. تُحفظ بيانات الطالب (الاسم، المحافظة، الجامعة، الكلية) في جدول `students`
4. الجلسة تبقى محفوظة → لا يحتاج تسجيل دخول مرة أخرى

هذا يحافظ على كل الميزات الحالية (اشتراكات، دفعات، اختبارات، إلخ) مع تبسيط التجربة بشكل كبير.
