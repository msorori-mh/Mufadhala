# سجل إصدارات تطبيق مُفَاضَلَة

## v5.2.0-stable (versionCode: 50200) — 2026-04-17
### تعزيزات أمنية حاسمة (Server-Side Enforcement)
- **محاكي الاختبار الحقيقي**: فرض حد المحاولة المجانية على السيرفر داخل `submit-exam` edge function. الحساب التجريبي (trial) لم يعد قادراً على تكرار الاختبار عبر التحايل على الواجهة.
- **المحادثة الذكية**: فرض حد 10 رسائل/يوم للمستخدم غير المدفوع داخل `chat` edge function. الرفض بـ 403 + `code: free_limit_reached` يحدث قبل استدعاء AI gateway فلا استهلاك رصيد.
- استبعاد صريح لمستخدمي trial من اعتبارهم مدفوعين عبر `.eq("status","active")` في كلا الـ functions.

### تحسينات تجربة المستخدم
- إضافة عداد ديناميكي "X/10 رسائل متبقية اليوم" في رأس ChatWidget مع تحويل لون الشارة إلى أحمر عند بقاء ≤2 رسالة.
- شريط تحذير تحت العداد عند بقاء ≤3 رسائل يشجّع على الاشتراك.
- شارة "المحاولة المجانية مستخدمة" في `PastExamModeMiniStats` تظهر للحساب التجريبي بعد استنفاد المحاولة المجانية في النماذج المجانية.

### أدوات الإدارة
- زر "اختبار سريع" في `/admin/ai-limits` يستدعي `generate-questions` بحساب الأدمن ويعرض القيم المُرجعة (limit, remaining, hasSubscription, code) للتحقق الفوري من سلامة الإعداد.

## v5.1.0-stable (versionCode: 50100) — 2026-04-17
### تحسينات وضع نماذج الأعوام السابقة
- ضغط بطاقات النماذج في `/past-exams` إلى سطر واحد (السنة + الاسم + المدة) — يُمكن رؤية 6-7 نماذج دفعة واحدة على الجوال بدلاً من 3.
- إضافة فاصل بصري متقطع خفيف بين السنوات المختلفة للحفاظ على إحساس التجميع دون استهلاك سطر كامل.
- ميزة "تمييز الفائز" في جدول المقارنة التفصيلية: إبراز الخلية الأفضل في كل صف بإطار ملوّن.

### تحسينات بنية
- إضافة `src/domain/version.ts` كمصدر وحيد لرقم الإصدار المعروض للمستخدم.
- عرض رقم الإصدار في صفحة الإعدادات بشكل ديناميكي.

## ملاحظات للبناء
عند بناء APK محلياً، تأكد من تحديث `android/app/build.gradle`:
```gradle
versionCode 50200
versionName "5.2.0"
```

ثم نفذ:
```bash
git pull
npm install
npm run build
npx cap sync android
cd android && ./gradlew assembleRelease
```
